# Evolution Toward Orion

The original G2D Researcher Cluster was an early step toward a more capable research infrastructure layer.

It was useful because it made the next system obvious.

That next direction became Orion.

---

## What the researcher cluster proved

The first cluster proved that separate research roles could create useful structure.

A baseline could be preserved.

A development branch could evolve.

A quantitative explorer could pursue fast hypothesis cycles.

A synthesis engine could merge and interpret findings.

That was enough to show that role-based research agents were worth pursuing.

---

## What the researcher cluster could not solve

The early cluster also exposed limits.

It did not have enough structure around:

- run identity
- evidence tracking
- memory boundaries
- workflow state
- artifact storage
- cross-agent coordination
- evaluation criteria
- report lineage

As the system grew, the problem shifted.

The question was no longer:

> Can an agent produce useful research output?

The question became:

> Can a research system coordinate roles, preserve evidence, and produce outputs that can be trusted later?

That question required a deeper architecture.

---

## Why Orion exists

Orion represents the deeper research-cluster direction that grew out of the first researcher lineage.

Where the original cluster explored role specialization, Orion points toward structured research operations:

- researcher roles
- synthesis roles
- critique roles
- run artifacts
- report generation
- evidence tracking
- reusable research workflows
- clearer boundaries between exploration and conclusion

In that sense, Orion is not simply a larger version of the first researcher cluster.

It is a response to what the first cluster taught.

---

## From agent variants to research operations

The first cluster used separate repos to represent different agent roles.

Orion moves the idea closer to an operating environment.

```text
Early researcher cluster:
  baseline + dev + variants

Orion direction:
  coordinated research roles + artifacts + evaluation + reporting
```

The shift is from:

> agents as separate experimental forks

Toward:

> agents as roles inside a governed research workflow

That distinction matters.

---

## Relationship to the control plane

The researcher cluster also helped clarify the need for a control plane.

As soon as workflows include multiple roles, the system needs answers to operational questions:

- who initiated the run?
- what task was assigned?
- what tools were available?
- which agent produced which artifact?
- what evidence supported the final report?
- what changed between runs?
- where should the output be stored?

These questions are not optional details. They are the foundation of trustworthy autonomous systems.

That is why the modern G2D direction emphasizes control-plane infrastructure, validation records, workflow ledgers, and reportable execution.

---

## What carries forward

The following ideas carried from the researcher cluster into later G2D thinking:

- preserve known-good reference states
- separate active development from stable artifacts
- specialize roles instead of forcing one general agent to do everything
- treat reports as system outputs, not decoration
- preserve evidence across runs
- make lineage readable
- design systems so that future operators can understand what happened

---

## What changed

The later direction became more disciplined.

Instead of only creating agent variants, the lab needed:

- stronger run records
- clearer tool boundaries
- durable artifacts
- more explicit evaluation
- safer operational routing
- better reporting layers
- a workflow-first architecture

That is the bridge from G2 Researcher v1.0 toward Orion and the broader Black Box Lab direction.

---

## Summary

The G2D Researcher Cluster was an origin point.

Orion is the evolution.

The cluster showed that role-based autonomous research was useful. Orion points toward making that research process structured, inspectable, and reliable enough to support serious lab work.
